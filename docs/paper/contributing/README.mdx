# 为 Paper 做贡献

感谢您对为 Paper 项目做贡献感兴趣！我们非常欢迎社区成员的贡献，无论是修复 bug、改进文档还是添加新功能。

## 开始之前

在开始为 Paper 做贡献之前，请确保：

1. 您已阅读并同意我们的[行为准则](https://docs.papermc.io/contributing/code-of-conduct)。
2. 您已经在 [Discord](https://discord.gg/papermc) 上加入了我们的社区。
3. 您对 Java 和 Minecraft 有基本的了解。

## 如何贡献

### 报告 Bug

如果您发现了 bug，请在我们的 [问题追踪器](https://github.com/PaperMC/Paper/issues) 上报告。在提交 bug 报告时，请：

- 使用搜索功能检查是否已有相同的问题报告
- 提供尽可能详细的信息，包括：
  - Paper 版本
  - 服务器日志
  - 重现步骤
  - 预期行为和实际行为

### 提交代码

1. Fork Paper 仓库
2. 创建您的功能分支 (`git checkout -b feature/amazing-feature`)
3. 提交您的更改 (`git commit -m '添加了一些很棒的功能'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 创建一个 Pull Request

### 改进文档

文档对于项目的可用性至关重要。如果您发现文档中有错误或需要改进的地方，请：

1. Fork 文档仓库
2. 进行必要的更改
3. 提交 Pull Request

## 开发指南

### 构建 Paper

请参阅我们的[构建指南](https://docs.papermc.io/paper/dev/building)了解如何在本地构建 Paper。

### 代码风格

我们使用特定的代码风格来保持代码库的一致性。在提交代码之前，请确保您的代码符合我们的[代码风格指南](https://docs.papermc.io/paper/dev/codestyle)。

### 测试

在提交 Pull Request 之前，请确保：

1. 您的代码通过了所有测试
2. 您为新功能添加了适当的测试
3. 您的更改不会破坏现有功能

## 获取帮助

如果您在贡献过程中需要帮助，可以：

- 在 Discord 的 `#paper-dev` 频道提问
- 在 GitHub 问题中寻求帮助
- 查阅我们的[开发者文档](https://docs.papermc.io/paper/dev)

## 贡献者许可协议

通过向 Paper 提交代码，您同意将您的贡献置于项目使用的许可证之下。

---

再次感谢您对 Paper 项目的贡献！您的参与对于项目的发展至关重要。
